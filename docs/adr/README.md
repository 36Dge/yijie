# Architecture Decision Records

ADR 用于记录跨仓、跨团队、长期影响的架构决策。

## 状态

- Proposed：提议中；
- Accepted：已接受；
- Deprecated：已废弃；
- Superseded：被后续 ADR 替代。

## 模板

```markdown
# ADR-000X: 决策标题

## 状态

Proposed | Accepted | Deprecated | Superseded

## 日期

YYYY-MM-DD

## 背景

说明为什么需要做这个决策。

## 决策

说明最终选择了什么方案。

## 备选方案

说明其它可选方案及优缺点。

## 影响

说明对仓库、团队、发布、安全、成本的影响。

## 风险

说明可能的问题。

## 后续动作

- [ ] 动作 1
```

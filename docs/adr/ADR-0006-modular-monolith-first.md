# ADR-0006: Go 业务后端优先采用模块化单体

## 状态

Accepted

## 日期

2026-07-04

## 背景

MVP 阶段业务域多，但团队需要快速验证闭环，过早微服务化会增加部署和联调成本。

## 决策

`yijie-api` MVP 阶段采用模块化单体，按认证、租户、店铺授权、任务、审批、审计、插件注册、知识库注册等模块分层。

## 备选方案

一开始拆微服务可以获得独立扩展能力，但会提高复杂度。

## 影响

- 统一业务主状态；
- 模块内部按 domain、application、infrastructure、interfaces 分层；
- 后续按压力和团队边界拆服务。

## 风险

模块边界不清会退化为大泥球，需要代码审查和 ADR 约束。
